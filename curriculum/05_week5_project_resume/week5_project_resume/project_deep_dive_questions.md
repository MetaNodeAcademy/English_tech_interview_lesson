# Week 5｜项目深挖、追问应对与简历优化讲义

---

## 本周目标（Why Week 5 matters）

Week 5 的核心不是“再背更多答案”，而是把你已经做过的项目，打磨成可以在技术面试里**稳定输出、可追问、可证明价值**的版本。

本周你要完成三件事：

1. 能讲清项目，不怕追问（技术细节 + 决策逻辑 + 结果复盘）
2. 能把项目写进英文简历（表达专业、重点突出、匹配岗位）
3. 为全真模拟面试做范围确认（题目边界清晰、准备更高效）

---

## Part 1｜项目深挖与追问设计（Zood）

技术老师会基于你的简历和目标岗位，帮你建立一套“可攻可守”的项目问答框架。

### 1) 项目深挖问题（Project Deep Dive）

用于确认你是否真的理解项目全貌，不只是“做过”。

**高频问题**
- “Can you walk me through this project end to end?”
- “What problem was this project trying to solve?”
- “What was your exact role in this project?”
- “Which part did you personally own?”

**回答要点（建议结构）**
- 背景：业务场景 + 问题痛点
- 目标：你负责解决什么
- 方案：核心架构与关键模块
- 个人贡献：你做了哪些动作
- 结果：指标变化 + 业务价值

---

### 2) 技术决策依据问题（Decision & Trade-off）

这类问题考察的不是“你会不会用某个技术”，而是“你为什么这样选”。

**高频问题**
- “Why did you choose this architecture / tech stack?”
- “What alternatives did you consider?”
- “What trade-offs did you make?”
- “If you had more time, what would you improve?”

**高分表达框架（英文）**
- “We chose A instead of B mainly because ___.”
- “The key trade-off was between ___ and ___.”
- “Given the timeline and system constraints, A was more practical because ___.”
- “If we iterate further, I’d improve ___ to better handle ___.”

---

### 3) 优化类 / Challenge 类 / Result 类追问

#### A. 优化类（Optimization）
**常见追问**
- “How did you identify the bottleneck?”
- “How did you validate the optimization result?”

**回答关键词**
profiling / baseline / benchmark / stress test / monitoring / rollback

#### B. Challenge 类（Difficulty Handling）
**常见追问**
- “What was the toughest challenge?”
- “What went wrong during implementation?”

**回答关键词**
root cause / mitigation / fallback plan / cross-team alignment / risk control

#### C. Result 类（Business Impact）
**常见追问**
- “How do you know your solution worked?”
- “What business impact did this bring?”

**回答关键词**
latency / throughput / SLA / stability / delivery efficiency / cost reduction

---

### 4) 岗位匹配表达重点（Role Match Framing）

同一个项目，针对不同岗位，讲法要不同。

- **后端岗位**：强调系统设计、性能、稳定性、接口治理
- **前端岗位**：强调体验优化、性能指标、工程化、协作交付
- **测试/QA岗位**：强调质量策略、自动化覆盖、缺陷闭环
- **DevOps/SRE岗位**：强调可观测性、发布可靠性、故障响应与恢复

**万能过渡句（英文）**
- “This experience is highly relevant to your role because ___.”
- “What I learned from this project can directly apply to ___.”

## Part 2｜项目讲述与简历打磨（Leon + Zood）

本周会把“会做项目”变成“会讲项目 + 会写简历”。

### 1) 输出个人项目经历英文优化版

目标：把原始中文经历转成可面试表达的英文版本（60 秒 / 90 秒 / 深挖版）。

**建议准备三档版本**
- **30 秒版**：快速说明项目类型与个人角色
- **60 秒版**：完整 STAR-lite 结构
- **90 秒版**：可承接深挖与追问

---

### 2) 梳理角色、决策、结果与亮点

每个项目至少补齐以下四块信息：

- **Role（角色）**：你是谁，你负责什么
- **Decision（决策）**：你做了哪些关键取舍
- **Result（结果）**：至少 1 个量化指标
- **Highlight（亮点）**：你跟同级候选人的差异点

**避免空话**
- 少说：“I participated in…”
- 多说：“I designed / implemented / led / optimized …”

---

### 3) 英文简历重点项目行修改与润色

重点不在“华丽词汇”，而在“信息密度 + 可验证性”。

**项目 bullet 推荐结构**
- Action verb + 你做了什么 + 用了什么 + 带来什么结果

**示例**
- “Redesigned the caching strategy using Redis and async preloading, reducing API P95 latency by 38% under peak traffic.”

---

### 4) 训练追问回答完整性与自然度

标准不是“背稿”，而是：
- 能听懂追问重点
- 能在 30–60 秒内组织答案
- 能补充细节但不跑题

**自检问题**
- 我是否明确说了我的个人贡献？
- 我是否解释了技术决策依据？
- 我是否给出可量化结果？
- 我是否能自然衔接下一个追问？

## Part 3｜模拟面试前准备

助教老师会与学员确认全真模拟时间，双师同步确定模拟范围。

### 你需要确认的三件事
1. **时间**：模拟具体日期与时段
2. **范围**：项目题 / 八股题 / 行为题占比
3. **重点**：优先攻克的薄弱模块（表达、深挖、追问）

- 个人项目经历英文表达优化版
- 简历重点项目润色版
- 全真模拟前问题范围确认

## 课后执行清单（建议）

1. 完成 1 个核心项目的 60 秒 + 90 秒英文讲述录音
2. 提交该项目的“追问清单 + 回答草稿（至少 8 题）”
3. 完成简历中 1–2 段重点项目 bullet 重写
4. 与助教确认模拟面试时间与范围

---

## 快速提醒（给学员）

Week 5 的关键不是“说得多”，而是“说得准”。

面试官真正想确认的是：
- 这个项目你是否真的做过；
- 你是否有独立决策和问题解决能力；
- 你的经验能否迁移到目标岗位。

把这三件事讲清楚，Week 5 就达标了。
